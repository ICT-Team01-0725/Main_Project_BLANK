[ 관리자페이지 : 사이트맵 ]
~ 계층에 따라서 TAB으로 구분하여 텍스트를 작성했습니다 ~
~ : 이후의 내용은 메뉴에 대한 설명입니다 ~ 

00 = 메인페이지
	: 관리자의 메인페이지의 경우에는
	  관리자페이지의 서브페이지 요소들을 배치함 
	  (GreenSpotFinder 때와 같은 화면 배치이면 될 것 같습니다)
	: 신고내역이나 문의글에 대한 알림이 있으면 좋을거 같습니다. (선생님의 권유사항)

01 = 대시보드
	1... 회원가입자 수 분석
		: 회원가입일 칼럼을 이용한 월별 가입자 수를 나타냄 (라인차트~월별, 분기별)

	2... 카테고리별 일별 클릭률 분석
		: 서브페이지별 클릭률을 나타냄 (파이차트)

	3... 사용자 피드백 분석
		: 전문가, 운영진에게 문의한 사용자 작성글에서 언급된 단어들의 빈도 표현 (워드클라우드)

	4... 사용자 검색기록 분석 ( 보류 )
		: 사용자 검색기록에서 자주 검색되는 키워드(약품명, 특정 제약회사 등) 표현

02 = 사용자 관리
	1... 일반 회원 관리
		1 ) 회원 목록 조회
		2 ) 회원 신고 내역 
		3 ) 경고 및 정지 

	2... 전문가 회원 관리
		1 ) 회원 목록 조회
		2 ) 회원 신고 내역
		3 ) 경고 및 정지
		4 ) 면허 승인 관리

03 = 관리자 관리
	1... 관리자 생성
		: 슈퍼 관리자가 일반 관리자들을 등록

	2... 관리자 관리
		: 슈퍼 관리자가 일반 관리자들의 정보를 수정, 삭제
		: 슈퍼 관리자가 일반 관리자들에 대한 영역별 권한 지정 (리스트형)
		: 슈퍼 관리자가 일반 관리자에게 권한 위임 ( 보류 )

04 = 페이지 관리
	: 페이지별 관리 개념에 DB에 대한 내용을 모두 포함합니다.
	: 관리자를 일종의 웹 이용자의 관점으로 보았을 때에는 웹개발자가 아니므로 
          기존에 보이는 사이트의 카테고리 순서에 따라 수정, 변경, 관리가 가능하도록 하는 것이 좋다고 생각됩니다.
	: 메뉴명, 카테고리명은 적극적으로 피드백 부탁드립니다.

	1... 의약품이란?
		: 1 ) ~ 4 ) : 텍스트, 이미지 수정 기능
		  5 ) :  API 관리, 탭메뉴 텍스트 수정 기능
		1 ) 의약품의 정의
		2 ) 의약품의 종류
		3 ) 의약품의 허가과정
		4 ) 신약과 제네릭
		5 ) 의약품 관련 법령

	2... 안전한 의약생활
		: 1 ) ~ 3 ) : DB에 대한 CRUD 기능
		1 ) 의약품 검색하기
		2 ) 부작용 검색하기
		3 ) 병용금기 검색하기

	3... 생활 속 의약
		: 1 ) ~ 2 ) : DB에 대한 CRUD 기능
		  3 ) : API 관리
		1 ) 약국 찾아보기
		2 ) 폐의약품 수거함 찾아보기
		3 ) 보도자료 찾아보기

	4... 팝업 관리
		1 ) 팝업 목록 (기존 등록된 팝업에 대한 수정, 삭제)
		2 ) 팝업 생성

05 = 커뮤니티 관리
	: 수업에서 이용한 기본적인 게시판 관리 기능 사용
	1... 공지사항
	2... 자주묻는 질문
	3... 전문가와의 상담
	4... 운영진에게 문의

06 = 보안 관리 ( 보류 )
	: 현재 스프링부트의 보안개념을 배우는 중이고, 
	 로컬 서버가 아닌 실서버에서 작동이 가능한 지에 대해서는 테스트 기간까지 투자해야하는 카테고리이기에
	 우선, '보안관리' 카테고리를 가장 마지막 카테고리로 두고, 
	 다른 기능이 모두 개발된 다음에 추가 작업을 하는 것으로 생각하고 있습니다.
	: 현재 1~5번의 카테고리에도 미약하나마 모두 보안 개념이 중첩되어 있으나,
	  6번 카테고리는 보안관리에 특화된 경우로 보시면 됩니다.
	1... 로그 관리
	2... 데이터 분석 
		: 시각화보다는 리스트 형태의 히스토리 조회
